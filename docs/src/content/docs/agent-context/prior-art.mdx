## Project history

Crosswalker started as a Python script (`frameworks_to_obsidian.py`) for importing cybersecurity frameworks into Obsidian. The Python tool successfully imports NIST 800-53, CIS Controls v8, MITRE ATT&CK/D3FEND/ENGAGE, and CRI Profile.

The Obsidian plugin extends this to a general-purpose import wizard with a UI, config system, and broader file format support.

## Existing tools in this space

### GRC platforms
- **ServiceNow GRC**, **Archer**, **LogicGate** — Enterprise GRC with proprietary databases
- **Drata**, **Vanta** — Automated compliance, SaaS-locked
- Trade-off: Full-featured but expensive, locked-in, rigid

### Knowledge/note tools
- **Obsidian + Dataview** — Flexible but no structured import
- **Notion databases** — Good structure but no graph navigation
- **Roam Research** — Graph-first but no filesystem

### Framework-specific tools
- **NIST OSCAL** — Machine-readable framework format (XML/JSON)
- **ATT&CK Navigator** — Visualization for MITRE ATT&CK
- **CIS Controls Navigator** — Interactive CIS tool

### Gap
No tool bridges structured framework data and a personal/team knowledge base with typed, queryable links. That's the Crosswalker niche.

## Related Obsidian plugins

| Plugin | Overlap | Difference |
|--------|---------|-----------|
| Dataview | Queries generated data | Doesn't import structured data |
| Folder Notes | Folder-as-note pattern | No import or generation |
| Metadata Menu | Property management | No bulk import from external files |
| Database Folder | Table views | No external data import wizard |
