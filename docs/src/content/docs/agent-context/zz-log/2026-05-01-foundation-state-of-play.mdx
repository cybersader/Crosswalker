## The web-of-webs framing

The general problem is **structured webs connecting to structured webs**.

Each [ontology](/crosswalker/concepts/terminology/#core-terms) Crosswalker handles — [NIST 800-53](/crosswalker/reference/registry/nist/), [CIS Controls](/crosswalker/reference/registry/cis/), MITRE ATT&CK, [SCF](/crosswalker/reference/registry/scf/) — is itself a structured web: typed nodes (controls, techniques, sub-techniques), typed edges (parent/child, references, mitigates), and edge metadata (confidence, scope, applicability). When we [import](/crosswalker/concepts/ontology-lifecycle/#import) one, we're [reifying that web into the filesystem](/crosswalker/agent-context/zz-log/2026-04-03-why-obsidian-why-files/) as folders, notes, frontmatter, and links.

Crosswalker's job is what happens *between* those webs. There are at least four kinds of inter-web connection in play, plus one optional intermediary:

```
                                  ┌────────────────────────┐
                                  │  evidence-vault web    │
                                  │ (your policies, audits,│
                                  │  technical docs, etc.) │
                                  └───────────┬────────────┘
                                              │
                                              │ evidence-link edges
                                              │ (junction notes, 13-field schema)
                                              ▼
   ┌──────────────┐    crosswalk      ┌────────────────┐    crosswalk     ┌──────────────┐
   │  NIST 800-53 │  ◄──────────────► │  optional      │ ◄──────────────► │  CIS Controls│
   │  ontology    │      edges        │  pivot web     │      edges       │  ontology    │
   │  web         │                   │  (a.k.a. spine,│                  │  web         │
   │              │                   │  hub, canonical│                  │              │
   │              │                   │  intermediate) │                  │              │
   └──────┬───────┘                   └────────────────┘                  └──────┬───────┘
          │                                                                      │
          │     lifecycle edges (between versions of the same web)               │
          └──────────────────────────────────────────────────────────────────────┘
                  9 atomic primitives: add / remove / rename / split / merge / ...
```

The four edge types and the optional pivot (a.k.a. **spine**, **hub**, **interlingua**, **canonical intermediate**) are the things Foundation is figuring out.

## The webs in play

| Web | What it is | Linked to |
|---|---|---|
| **Source-ontology web** | An imported framework — folders + notes + typed links carrying the structure of NIST/CIS/MITRE/etc. into the vault | [Ontology lifecycle](/crosswalker/concepts/ontology-lifecycle/) · [Framework crosswalks](/crosswalker/agent-context/framework-crosswalks/) · [Why files](/crosswalker/agent-context/zz-log/2026-04-03-why-obsidian-why-files/) |
| **Evidence-vault web** | The user's own notes — policies, audit reports, configuration evidence, interview notes | [Evidence mapping for GRC teams](/crosswalker/getting-started/grc-teams/) · [Link metadata system (historical)](/crosswalker/agent-context/link-metadata-system/) |
| **Optional pivot web** (a.k.a. spine, hub, interlingua, canonical intermediate) | A canonical intermediate ontology between source webs — synthetic (computed via FCA), inherited (SCF, OSCAL catalog), or handcrafted | [Challenge 06](/crosswalker/agent-context/zz-challenges/archive/06-synthetic-spine/) · [Schema matching: pivot approach](/crosswalker/concepts/schema-matching/#the-interlingua--pivot-approach) |
| **Lifecycle (cross-version) graph** | Versions of an ontology over time, connected by structural-change atoms (add, remove, rename, split, merge, deprecate, etc.). Not a separate "meta-web" — the lifecycle problem applies to *every* web above. | [Ontology evolution](/crosswalker/concepts/ontology-evolution/) · [Atomic operations research](/crosswalker/agent-context/zz-log/2026-04-09-atomic-operations-research/) · [EvolutionPattern draft](/crosswalker/agent-context/zz-log/2026-04-03-evolution-pattern-taxonomy-draft/) |

Every Foundation question is a question about one of these webs or the connections between them.

| **Lifecycle edges** | between versions of the same source-ontology web | [9 atomic operations + 4 composites](/crosswalker/agent-context/zz-log/2026-04-09-atomic-operations-research/), provably complete per graph edit distance literature | [Atomic operations research](/crosswalker/agent-context/zz-log/2026-04-09-atomic-operations-research/) · [Primitives depth](/crosswalker/agent-context/zz-log/2026-04-09-primitives-depth-and-pluggable-layers/) |
