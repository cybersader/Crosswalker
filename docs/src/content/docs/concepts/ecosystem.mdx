## Obsidian plugins

Crosswalker is designed to complement existing Obsidian tools:

| Plugin | Relationship |
|--------|-------------|
| **[Dataview](/crosswalker/concepts/metadata-ecosystem/)** | Query Crosswalker-generated frontmatter |
| **[Obsidian Bases](/crosswalker/concepts/metadata-ecosystem/)** | Create table/card views of framework data |
| **Folder Notes** | Treat framework folders as navigable notes |
| **Templater** | Customize generated note templates |
| **QuickAdd** | Create macros for evidence linking |

## GRC tools

> [!tip] Metadata ecosystem
> For a detailed analysis of how Obsidian's metadata landscape is evolving (Properties, Dataview, Bases, Datacore) and what it means for Crosswalker, see [Obsidian metadata ecosystem](/crosswalker/concepts/metadata-ecosystem/).

Crosswalker doesn't replace GRC platforms — it provides a knowledge-layer complement:

- **Import from**: [NIST](/crosswalker/reference/registry/nist/), [CIS](/crosswalker/reference/registry/cis/), [MITRE](/crosswalker/reference/registry/mitre/), [ISO](/crosswalker/reference/registry/iso/), [CRI](/crosswalker/reference/registry/cri/), [FFIEC](/crosswalker/reference/registry/ffiec/), and any CSV/XLSX/JSON source
- **Export to**: Planned [OSCAL](/crosswalker/reference/registry/oscal/) export for interoperability (see the [crosswalk edge semantics commitment](/crosswalker/agent-context/zz-log/2026-04-10-foundation-research-synthesis/#edge-semantics-should-be-set-theoretic-not-skos-ish) and the [evidence-link edge model synthesis](/crosswalker/agent-context/zz-log/2026-04-10-evidence-link-edge-model-synthesis/) for the two edge-type export targets)
- **Bridge**: Plain-text notes that can feed into any system

### CISO Assistant
Open-source GRC tool supporting 80+ frameworks (NIST CSF, ISO 27001, SOC2, CIS, PCI DSS, NIS2, CMMC, GDPR, HIPAA, and more). Key features:
- **Decoupling principle** — define controls once, reuse across frameworks
- **Auto-mapping** — automatic crosswalks between supported standards
- **API-first** — programmatic access for integration
- [GitHub](https://github.com/intuitem/ciso-assistant-community)

Where Crosswalker differs: CISO Assistant is a full GRC database; Crosswalker brings framework data *into your knowledge base* as plain-text notes you own.

### SCF with STRM
The [Secure Controls Framework](/crosswalker/reference/registry/scf/) uses [Set Theory Relationship Mapping (STRM)](/crosswalker/reference/registry/strm/) — a mathematical approach to crosswalking frameworks. The [STRM Excel bundle](https://securecontrolsframework.com/strm-bundle/) is one of the most comprehensive mapping datasets available. Crosswalker's [04-10 foundation synthesis](/crosswalker/agent-context/zz-log/2026-04-10-foundation-research-synthesis/#edge-semantics-should-be-set-theoretic-not-skos-ish) commits to STRM's 5-relationship vocabulary (paired with [SSSOM](/crosswalker/reference/registry/sssom/) as the metadata envelope) as the required `predicate_id` for every crosswalk edge.

## Design philosophy

1. **Plain text first** — Everything in markdown, YAML, and JSON
2. **Framework agnostic** — Easy to add new frameworks
3. **Tool-agnostic outputs** — Works with Bases, Dataview, or nothing
4. **No lock-in** — Standard Obsidian files, delete the plugin and your data remains
